### Detailed Findings

| # | Severity | CVE | Finding | Affected Package | Vulnerable Version | Path | CVSS Score | Fix |
|---|----------|-----|---------|------------------|-------------------|------|------------|-----|
| 1 | **HIGH** | CVE-2026-33671 | ReDoS vulnerability via extglob quantifiers | picomatch | 2.3.1 | eslint-config-next → @next/eslint-plugin-next → fast-glob → micromatch → picomatch | 7.5 | Upgrade picomatch to ≥2.3.2 |
| 2 | **HIGH** | CVE-2026-33671 | ReDoS vulnerability via extglob quantifiers | picomatch | 4.0.3 | eslint-config-next → eslint-import-resolver-typescript → tinyglobby → picomatch | 7.5 | Upgrade picomatch to ≥4.0.4 |

| 4 | **HIGH** | Not disclosed | Arbitrary File Read via Vite Dev Server WebSocket | vite | 8.0.0 | vite (direct dev) | Not disclosed | Upgrade vite to ≥8.0.5 |
| 5 | ADVISORY | CVE-2026-33672 | Method Injection in POSIX Character Classes causes incorrect Glob Matching | picomatch | 2.3.1 | eslint-config-next → @next/eslint-plugin-next → fast-glob → micromatch → picomatch | 5.3 | Upgrade picomatch to ≥2.3.2 |
| 6 | ADVISORY | CVE-2026-33672 | Method Injection in POSIX Character Classes causes incorrect Glob Matching | picomatch | 4.0.3 | eslint-config-next → eslint-import-resolver-typescript → tinyglobby → picomatch | 5.3 | Upgrade picomatch to ≥4.0.4 |
| 7 | ADVISORY | CVE-2026-33532 | Stack Overflow via deeply nested YAML collections | yaml | 2.8.2 | vite → yaml | 4.3 | Upgrade yaml to ≥2.8.3 |

**Advisory details:**

- **CVE-2026-33671 (GHSA-c2c7-rcm5-vvqj):** Picomatch compiles certain extglob patterns (e.g., `+(a|aa)`) into regexes with catastrophic backtracking. Applications processing untrusted glob patterns face DoS risk. Not applicable to this project—picomatch is used internally by build tooling with developer-controlled patterns.

- **CVE-2026-33672 (GHSA-3v7f-55p6-f55p):** POSIX bracket expressions like `[[:constructor:]]` reference inherited method names, causing incorrect glob matching. No RCE risk. Impact limited to logic errors in glob filtering. Not applicable to this project—picomatch usage is internal to build tools.

- **CVE-2026-33532 (GHSA-48c2-rrv3-qjmp):** YAML parser throws `RangeError` on deeply nested flow sequences (~2–10 KB input). Affects `YAML.parse()`, `YAML.parseDocument()`, `YAML.parseAllDocuments()`. Vite uses yaml for config parsing. Risk is limited to development environment—no user-supplied YAML is parsed.

- **CVE-2026-39365 (GHSA-v2wj-q39q-566r):** Vite dev server's `server.fs.deny` protection can be bypassed with crafted query strings, allowing arbitrary file read. Affects vite 8.0.0 through 8.0.4. Development-only impact—production builds (Next.js) do not run Vite dev server.

- **Vite WebSocket Advisory:** Vite dev server WebSocket endpoint allows arbitrary file reads. Development-only impact—production builds do not expose this endpoint.
